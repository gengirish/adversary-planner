## Key Design Decisions

- **Stateless API**: All campaign state lives client-side (localStorage) to work on Vercel serverless (no persistent disk).
- **Client sends state with every request**: API reconstructs the BayesianPlanner from technique_states on each call.
- **CLI shares core logic**: Both CLI and web API import from `adversary_planner` package.
